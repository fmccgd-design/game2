# 아레나 슈팅 대결

브라우저에서 친구와 함께 즐기는 실시간 2~5인 온라인 슈팅 게임입니다. 서버 없이 P2P(PeerJS/WebRTC)로 연결되며, HTML 파일 하나로 동작합니다.

## 주요 특징

- **최대 5인 대전**: 방장(P1) + 참가자 최대 4명
- **대기실**: 방을 만들면 바로 대기실로 이동해 참가자 목록, 방장이 고른 경기장/모드, 각자 선택한 능력을 실시간으로 확인 가능
- **경기장 3종**
  - 사각형 경기장
  - 원형 경기장
  - 팔각형 경기장
- **모드 3종**
  - 데스매치 (FFA): 혼자 살아남으면 승리, 부활 없음
  - 팀전: 2팀으로 나뉘어 대결(아군 오사 없음), 부활 없음
  - 거점 점령전: 중앙 거점을 혼자 점유해 점수를 쌓고 먼저 20점을 획득하면 승리, 죽으면 3초 후 부활
- **능력 5종** (경기 전 1개 선택, 대시는 전원 기본 사용 가능)
  - 실드: 2초간 무적
  - 순간이동: 바라보는 방향으로 짧게 블링크
  - 회복: 체력 25 회복
  - 스피드 부스트: 3초간 이동 속도 증가
  - 지뢰 설치: 밟으면 폭발하는 지뢰 설치

## 실행 방법

`file://`로 파일을 더블클릭해서 열면 WebRTC 연결이 차단될 수 있어 정상적으로 동작하지 않습니다. 아래 중 한 가지 방법으로 로컬 웹서버를 띄워서 접속해주세요.

```bash
# 이 파일이 있는 폴더에서 실행
python3 -m http.server
```

그 다음 브라우저에서 `http://localhost:8000` 으로 접속합니다. 같은 방식으로 두 대의 기기(또는 두 개의 브라우저 창)에서 각각 접속해 방을 만들고 참가하면 됩니다.

인터넷을 통해 원격지 친구와 플레이하려면 GitHub Pages 같은 정적 호스팅에 `index.html`을 올려서 URL을 공유하면 됩니다.

## 플레이 방법

1. 한 명이 **방 만들기**를 눌러 연결 코드를 생성하고, 이 코드를 친구에게 공유합니다.
2. 나머지 인원은 **참가하기**를 눌러 코드를 입력하고 연결합니다. (최대 4명까지 추가 참가 가능)
3. 대기실에서:
   - 방장은 경기장과 모드를 선택합니다.
   - 각 플레이어는 자신의 능력을 1개 선택합니다.
   - 팀전 모드에서는 자신의 팀 배지를 눌러 팀을 바꿀 수 있습니다.
4. 모든 플레이어가 능력을 선택하면 방장이 **게임 시작**을 누를 수 있습니다.
5. 경기가 끝나면 방장만 **다시 시작**을 눌러 같은 설정으로 재대결할 수 있습니다.

## 조작법

| 동작 | 키/입력 |
|---|---|
| 이동 | W A S D |
| 조준 | 마우스 |
| 사격 | 마우스 클릭 |
| 강공격 | Q |
| 대시 (기본 능력) | Shift |
| 선택한 능력 발동 | E |

## 기술 스택

- 순수 HTML/CSS/JavaScript, 외부 프레임워크 없음
- 네트워킹: [PeerJS](https://peerjs.com/) (WebRTC 기반 P2P)
- 렌더링: HTML5 Canvas 2D
- 구조: 방장(호스트)이 시뮬레이션을 전담하는 호스트 권위(host-authoritative) 방식으로, 참가자는 입력만 전송하고 매 프레임 스냅샷을 받아 렌더링합니다.

## 알려진 제한사항

- 방장이 게임 중 접속을 끊으면 세션이 종료됩니다(방장이 시뮬레이션을 전담하기 때문).
- 네트워크 지연(핑)이 큰 환경에서는 다소 끊김이 있을 수 있습니다.
- 참가자가 경기 중 나가면 해당 캐릭터는 즉시 탈락 처리됩니다.
